Для регламентации ввода планов на период планирования объектом управления в конфигурации используются модели планирования, задаваемые в одноименном справочнике. Каждый элемент справочника «Модель планирования» содержит в себе период планирования, должность и квалификацию объекта управления, сезон, набор целевых показателей с весами, набор стандартов с весами.В справочник «Модель планирования» заносятся записи с набором целевых показателей, которые будут автоматически подставляться при занесении новых планов для объектов управления. В справочнике есть предопределенный элемент – базовая модель, в котором заданы наиболее используемые и общие показатели (SMART-задачи и стандарты. Базовая модель по умолчанию применяется ко всем вновь созданным элементам структуры компании, до тех пор, пока им не будет присвоена какая-либо другая модель.Следует отметить, что при заполнении моделей планирования: определяются и плановые данные, автоматически попадающие в конкретные планы пользователей (т.е. важен выбор значений по умолчанию, что влияет на эффективность работы в системе)задаются все ограничения действий с данными по объектам управления.Остановимся подробнее на структуре данных, задаваемых в элементах справочника «Модель планирования».На первой вкладке задаются целевые показатели пользователя, задаётся их вес (колонка «Вес») и значение, которое будет соответствовать полному, 100%-му выполнению плана по этому показателю (колонка «План»). При этом в список могут вводиться 2 предопределенных вида целевых показателей, связанных с совершенно другими  структурами данных: это показатели «SMART-задачи» и «Стандарты». Пользователь должен определить их вес (используемый при подсчете общего коэффициента результативности по пользователю), а в качестве плана автоматически подставляется значение 100 (т.е. поскольку выполнение задач и стандартов с учетом различных факторов в итоге приводится к процентам, то и полное выполнение плана по ним в этом случае соответствует 100 процентам).
